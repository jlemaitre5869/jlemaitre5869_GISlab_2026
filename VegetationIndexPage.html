Vegetation Index Maps

The purpose of this project is to create a Vegetation Index. Therefore, I have decided to create a Vegetation Index Maps of Angola in 2018 and 2024 to see whether deforestation has occurred. I chose Angola because it is an area which, acording to the UN, is experiencing a lot of deforestation. These map will therefore address SDG 15, life on land, and 13, climate action. Deforestation is a severe problem as not only does it affect ecosystems, but also climate change.

Project Overview

Before creating the maps which look at vegetation index we first need to understand what an orthomosaic is. An othomosaic is the output when multiple images, which have been taken from arial drone or satellites, are overlayed to produce a continuous image of section of a map.

As mentioned, above the multiple overlayed images are taken by satellite imaging, specifically multispectral satellite imagery. Multispectral satellite imagery is the process which uses multiple specific wavelength to capture data. It does this by only recording red, green and blue. Through this process it can capture specific layers of information, such as, plant health, moisture in soil and so on. This imaging technique is thus very important as it can help determine which region within a country have dry soil or unhealthy plant life. This can lead to government targetting specific areas and improving, for example, irrigation systems. This satellite imaging is also relevant to sustainable development. By looking at the examples mentioned above one can see that the layers already could be useful for goal 15, 3 and 6.

An example of a multispectral imaging satellite is Sential 2. Sential 2 is a passive satellite and its payload is both a multi-spectral instrument and an optical communication payload. It runs by measuring the naturally reflected sunlight from the earth surface instead of producing its own energy signal.

Now that we understand how to create a vegetation index map using both orthomosaics and satellites, we need to understand what the index is. The vegetation index is a mathematical formula applied to orthomosaics to determine health, density and vigor of the plants in that area. They are used for soil management, water management, precision agriculture and climate monitoring. A specific example of a vegetation index is the NDVI. The NDVI or the normalized difference vegetation index is a method to measure the plants of a given area greenness and health. The index does this by assessing the difference between near-infrared and red light reflectance. This is because plants which have healthy chlorophyll reflect more near-infrared and absorb more red light. This is why your choice of time during the year for your orthomosaic map is important, because if one is looking at NDVI the plants phenology changes with seasons.

Methods

To make an NDVI map on QGIS we first need to attain the orthomosaic. However, as one can imagine clouds will distort with the product of an orthomosaic. So, to make a cloudless composite orthomosaic from open data one can use an program such as Google Earth Engine. To get a cloudless composite one can code this to find the ariel images without them. Then once you have written your code you can highlight your region of interest using the polygon function. This is basically drawing a box around the area you want ariel images for. Once you have done this, you determine the time period of interest and you can run your code. This should render the cloudless orthomosaic which you can then download.

Results & Findings

The results from these maps were that there was less healthy vegetation in 2024 compared to 2018. This can be seen by comparing both NDVI maps form each year. This can also be seen by looking at the difference map which shows value between 0-0.3 which means that the NDVI was larger in 2018 than 2024. This implies that deforestation has occurred in this region. Therefore planning recommendation could be reforestation.

Reflection

In this project I have learnt how to create an orthomosaic in Google Earth Engine and make a Vegetation Index Map in QGIS. A limitation to this project is that QGIS struggled with the size of the orthomosaic file because my area was to big. This meant that I had to redo my ortomosaic again for QGIS to work. Therefore, one lesson I have learnt is always go for the smallest area when downloading data. Another, limitation to this project was despite me changing the area size my QGIS still struggled to download my final project maps. This meant that I had to screenshot my two true colour maps. This means that my scale might not be as accurate and the quality of the maps are poor. So for future improvement, it is best to choose a small area or use a computer with more power.

To improve my project for the future, it would have been nice to look at different areas in Angola to see if they experienced similar deforestation patterns between 2018-2024. It would have also been nice to see each year in between 2018-2024 to see the deforestation progress.
